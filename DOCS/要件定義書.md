# 要件定義書

**プロジェクト名:** SES-RAG-DEMO
**最終更新日:** 2026-05-19

---

## 1. 目的と背景

- SES(System Engineering Services)業界では、案件要件とエンジニアのマッチングが営業担当の手作業に依存することが多く、属人化と業務非効率の原因となっている。
- 本システムはRAG(Retrieval-Augmented Generation)技術を活用し、案件要件とエンジニアのスキルプロフィールを意味的な類似性に基づきマッチングする。
- 「AIによる意思決定支援」を示すデモとして位置づける。

## 2. 課題定義(SESマッチングの現状課題)

- **現状:** 営業担当がエンジニアのスキルシートと案件要件をスプレッドシートや目視で比較している。
  - **課題1(検索精度):** キーワードによる完全一致では表記揺れ(例:「Javaエンジニア」と「Spring Framework経験」)を吸収できず、マッチング機会を取りこぼす。
  - **課題2(説明可能性):** なぜその候補を推薦するのかという根拠の言語化に高い認知コストがかかる。
  - **課題3(スケーラビリティ):** 候補エンジニア数が増えるにつれて手動比較のコストが線形に増加する。

本システムはEmbeddingによる意味的検索で課題1と課題3に対処し、LLMによる理由生成で課題2に対処する。

## 3. システム概要

- 案件要件のテキストを入力として受け取り、意味的に類似度の高いエンジニア候補を上位k件取得し、それぞれについてLLMが自動でマッチング理由を生成する。
- バックエンド:LangChain + ChromaDB + OpenAI API
- フロントエンド:Streamlit(Webブラウザ上で動作)
- データ:架空の案件要件20件(`jobs.json`)と架空のエンジニア20件(`engineers.json`)
- デプロイ:Streamlit Cloud(GitHub連携)

## 4. 機能要件

### 4.1 インデックス構築機能

- `engineers.json` を読み込み、各エンジニアのプロフィールをEmbeddingに変換する。
- 変換したベクトルをChromaDBのコレクションに登録する(初回構築・上書きの双方に対応)。
- StreamlitサイドバーのボタンUIから実行可能。
- 完了後、登録件数と結果をUIに表示する。

### 4.2 エンジニア検索機能

- 入力された案件要件テキストをOpenAI Embedding APIでベクトル化する。
- ChromaDBに保存されたエンジニアプロフィールベクトルとの類似度を計算する。
- コサイン類似度上位k件(デフォルト:3件)を返却する。
- 検索結果にはエンジニア名・スキル・類似度スコアを含める。

### 4.3 マッチング理由生成機能

- 上位k件の候補エンジニア情報と案件要件をプロンプトに組み込む。
- gpt-4o-miniに送信し、候補ごとのマッチング理由を生成する。
- 理由は2〜3項目の箇条書きとし、根拠を明示する。
- 採用判断を自動化するものではなく、人間の意思決定を補助する説明文として提示する。

## 5. 非機能要件

### パフォーマンス

| 処理 | 目標値 |
|---------|--------|
| 検索(Embedding生成 + ChromaDBクエリ) | 3秒以内 |
| LLMによる理由生成(gpt-4o-mini) | 10秒以内 |
| インデックス構築(20件) | 30秒以内 |

### セキュリティ

- OpenAI APIキーは環境変数(`.env`)で管理し、ハードコードしない。
- `.env` ファイルは `.gitignore` に登録し、GitHubへのコミットを防ぐ。
- Streamlit Cloud上では `st.secrets` でシークレットを管理する。

### 可用性

- Streamlit Cloud無料プランの制約(スリープモード等)を許容する。
- ローカル環境でも同等の動作を保証する。

## 6. 制約事項

| 項目 | 内容 |
|------|--------|
| 言語 | Python 3.14 |
| LLM | OpenAI gpt-4o-mini(コスト最小化のためminiを選定) |
| Embedding | text-embedding-3-small(同上の理由) |
| ベクトルDB | ChromaDB(ローカルファイル保存) |
| データ量 | 案件20件・エンジニア20件(デモ用に固定) |
| 予算 | OpenAI API(従量課金):インデックス構築 約$0.0001(初回のみ)、1クエリ約$0.0004(100クエリで約$0.04)。デモ規模では無視できる水準。(2026-05-10時点の単価:text-embedding-3-small $0.02/1Mトークン、gpt-4o-mini 入力 $0.15・出力 $0.60/1Mトークン)。Streamlit Cloud無料プラン(スリープモード許容)。 |
| サンプルデータ | 実在企業名・実在個人名は使用しない(架空のみ) |

## 7. 用語集

| 用語 | 定義 |
|------|-----------|
| SES(System Engineering Services) | クライアント先にエンジニアを派遣する準委任・請負契約形態のIT人材サービス。 |
| RAG(Retrieval-Augmented Generation) | 外部知識を検索してLLMの生成を補強する技術。 |
| Embedding | テキストを意味を保持した数値ベクトルに変換する処理。 |
| ChromaDB | ローカルで動作するオープンソースのベクトルDB。 |
| コサイン類似度 | 2つのベクトル間の角度から意味的な近さを測る指標(範囲:0〜1)。 |
| Top-k | 類似度上位k件を返却する検索設定。本システムのデフォルトは3。 |
