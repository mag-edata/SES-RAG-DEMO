**バージョン:** 1.0

## 1. 画面一覧

| No. | 画面名 | 説明 |
|-----|-------------|-------------|
| 1 | メイン画面 | 案件要件入力とマッチング結果表示 |
| 2 | サイドバー | インデックス構築操作パネル |

> 本システムはシングルページアプリケーション(SPA)として実装する。画面遷移は発生せず、すべての操作は単一画面内で完結する。

## 2. 画面設計

### 2.1 メイン画面(案件入力 / 結果表示)

**レイアウト**

- ページタイトル:`ITエンジニア × 案件要件マッチング(RAGデモ)`
- サブタイトル:システムの概要を1〜2行で表示

**入力エリア**

- ラベル:「案件要件を入力してください」
- コンポーネント:`st.text_area`(高さ約400px)
- プレースホルダー:入力例をヒントとして表示
- ボタン:「マッチング実行」(`st.button`)

**結果表示エリア**

- 検索実行後に表示(ローディングスピナー付き)
- 候補エンジニアごとに `st.expander` で表示
- 表示項目:順位・エンジニア名・スキル概要・類似度スコア・マッチング理由
- 理由は箇条書き(2〜3項目)で表示

### 2.2 サイドバー(インデックス構築)

**レイアウト**

- ヘッダー:「管理メニュー」
- 説明文:インデックス構築に関する簡単な説明(1行)
- ボタン:「インデックス構築」(`st.button`)
- 実行後の表示:成功メッセージ(登録件数)またはエラーメッセージ
- ChromaDB状態表示:現在の登録件数(`st.metric`)

## 3. 入出力定義

### 3.1 入力(案件要件テキスト)

| 項目 | 内容 |
|-------|--------|
| 項目名 | 案件要件テキスト |
| 入力方法 | テキストエリアへの手入力 |
| データ型 | 文字列(`str`) |
| 最大長 | 1000文字(UI制約なし、API上限に準拠) |
| 必須/任意 | 必須 |
| バリデーション | 空文字の場合はエラーメッセージを表示し処理を中止 |

**入力例**

```
Pythonによるバックエンド開発案件。FastAPIとAWSの経験必須。
リーダー経験があると望ましい。期間は3か月以上を想定。
```

### 3.2 出力(上位k件の候補エンジニア + 理由)

出力形式:Streamlit UI上でのリスト表示

| 項目名 | データ型 | 説明 |
|------------|-----------|-------------|
| rank | int | 候補順位(1始まり) |
| engineer_name | str | エンジニア名 |
| skills | str | スキル一覧(カンマ区切り、例:"Python, AWS, Docker") |
| similarity_score | float | 類似度スコア(範囲 [0.0, 1.0])。`max(0.0, 1 - cosine_distance)` で算出。`cosine_distance` はChromaDB(`hnsw:space="cosine"` 設定)が返す値。数学的にはコサイン類似度と等価で、浮動小数点誤差を吸収するため0.0で下限クランプする。値が大きいほど類似度が高い。 |
| matching_reason | str | LLMが生成したマッチング理由 |

**出力例(UI表示イメージ)**

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
候補 1:田中 一郎(類似度: 0.87)
スキル:Python, FastAPI, AWS, Docker
マッチング理由:
- 必須のFastAPI経験を有しており、即戦力として期待できる。
- AWSインフラ3年の経験があり、インフラ面でも付加価値を提供できる。
- リードエンジニア経験を有しており、リーダー要件を満たす。
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 4. エラーメッセージ一覧

| コード | 発生条件 | 表示メッセージ |
|------|-------------------|-----------------|
| E001 | 入力テキストが空 | 「案件要件を入力してください。」 |
| E002 | ChromaDBにデータが未登録 | 「インデックスが構築されていません。サイドバーから構築してください。」 |
| E003 | OpenAI APIキー未設定 | 「APIキーが設定されていません。.env ファイルを確認してください。」 |
| E004 | OpenAI API呼び出し失敗 | 「API呼び出しに失敗しました。しばらく待ってから再試行してください。」 |
| E005 | engineers.json 読み込み失敗 | 「データファイルの読み込みに失敗しました。ファイルパスを確認してください。」 |
| E006 | 検索結果が0件 | 「該当するエンジニアが見つかりませんでした。要件を変更して再度お試しください。」 |

> エラーは `st.error()` コンポーネントで赤色表示する。

## 5. API インターフェース(OpenAI API呼び出し仕様)

### 5.1 Embedding API(テキストのベクトル化)

| 項目 | 内容 |
|------|--------|
| エンドポイント | `POST https://api.openai.com/v1/embeddings` |
| モデル | `text-embedding-3-small` |
| 入力 | テキスト文字列(案件要件またはエンジニアプロフィール) |
| 出力 | 1536次元の浮動小数点ベクトル |
| 呼び出しタイミング | インデックス構築時(engineers.json の全件、最大20回)/ マッチング実行時(入力テキスト、1回) |
| エラーハンドリング | `RateLimitError`・`AuthenticationError` を捕捉 → E004 / E003 に変換 |

### 5.2 Chat Completions API(理由生成)

| 項目 | 内容 |
|------|--------|
| エンドポイント | `POST https://api.openai.com/v1/chat/completions` |
| モデル | `gpt-4o-mini` |
| max_tokens | 800 |
| temperature | 0.3(再現性を優先) |
| 入力(system) | マッチング理由生成のためのロール指示と出力フォーマット指定 |
| 入力(user) | 案件要件テキスト + 候補エンジニア情報(JSON形式) |
| 出力 | マッチング理由テキスト(日本語・箇条書き) |
| 呼び出しタイミング | マッチング実行時(上位k件に対して最大k回、もしくは1回のバッチ呼び出し) |
