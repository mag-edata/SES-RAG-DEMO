## 2. モジュール構成

### 2.1 `rag/embedder.py`

**役割:** エンジニアデータからEmbeddingを生成しChromaDBに登録する

**主要関数**

| 関数 | シグネチャ | 説明 |
|----------|-----------|-------------|
| `load_engineers` | `(filepath: str) -> list[dict]` | `engineers.json` を読み込みリストとして返却。 |
| `build_profile_text` | `(engineer: dict) -> str` | エンジニアの辞書フィールドを単一の文字列に連結(検索対象テキストとして使用)。例:`"名前: 田中 一郎  スキル: Python, AWS  経験年数: 5 ..."` |
| `build_index` | `(engineers: list[dict] = None, collection_name: str = "engineers") -> int` | 全エンジニアのプロフィールテキストをEmbedding化しChromaDBに登録する。`engineers` を省略した場合は内部で `load_engineers()` を呼び出す。既存コレクションを削除して再構築する(冪等性を保証)。登録件数を返却する。 |

**外部依存:** `openai.OpenAI`、`chromadb`、`json`、`os`

### 2.2 `rag/retriever.py`

**役割:** 案件要件テキストに基づくエンジニアの類似度検索

**主要関数**

| 関数 | シグネチャ | 説明 |
|----------|-----------|-------------|
| `get_collection` | `(collection_name: str = "engineers") -> chromadb.Collection` | ChromaDBコレクションを取得して返却する。 |
| `search_engineers` | `(query_text: str, top_k: int = 3) -> list[dict]` | クエリテキストをEmbedding化し、コレクションから top_k 件を検索する。返却形式:`[{"id": str, "document": str, "distance": float, "metadata": dict}]`。 |
| `retrieve` | `(query: str, k: int = 3) -> list[dict]` | `search_engineers()` のラッパー。ChromaDBのコサイン距離を類似度スコア(`1 - distance`)に変換し、`chain.py` から利用しやすい `[{"id", "name", "skills", "experience_years", "score", ...}]` 形式で返却する。 |

**外部依存:** `openai.OpenAI`、`chromadb`、`rag.embedder`

### 2.3 `rag/chain.py`

**役割:** LangChain RAG Chain によるマッチング理由生成

**主要関数**

| 関数 | シグネチャ | 説明 |
|----------|-----------|-------------|
| `build_prompt` | `(job_text: str, candidates: list[dict]) -> str` | 案件要件と候補エンジニアリストからLLM向けのプロンプト文字列を生成する。 |
| `generate_reasons` | `(job_text: str, candidates: list[dict]) -> str` | プロンプトを gpt-4o-mini に送信し、マッチング理由テキストを返却する。LangChain の `ChatOpenAI` と `PromptTemplate` を使用。 |
| `run_rag` | `(job_description: str, k: int = 3) -> list[dict]` | **メインエントリーポイント。** RAGパイプライン全体を実行する:`retrieve()` を呼び出した後 `generate_reasons()` を実行し、候補ごとの理由テキストにパースして `[{"engineer": dict, "reason": str}]` を返却する。`app.py` から呼び出される。 |

**外部依存:** `langchain_openai.ChatOpenAI`、`langchain_core.prompts.PromptTemplate`、`os`

### 2.4 `app.py`

**役割:** Streamlit UIのエントリーポイント

**処理フロー**

1. ページ設定とタイトル表示
2. 環境変数の読み込み(python-dotenv)
3. サイドバー:インデックス構築ボタン → `embedder.build_index()` を呼び出し
4. メイン:テキストエリア入力 → マッチング実行ボタン
5. ボタン押下時:`chain.run_rag()` → `retriever.retrieve()` → `chain.generate_reasons()` → 結果表示

**外部依存:** `streamlit`、`dotenv`(python-dotenv)、`rag.embedder`、`rag.retriever`、`rag.chain`

## 3. データフロー

### インデックス構築フロー

1. `app.py`(サイドバー):ボタン押下イベントを受け取る
2. `embedder.load_engineers()`:`data/engineers.json` を読み込む(20件)
3. `embedder.build_profile_text()`:各エンジニアを文字列に変換
4. OpenAI Embeddings API:テキスト → 1536次元ベクトル(20回のAPI呼び出し)
5. ChromaDB:ベクトル・メタデータ・ドキュメントを "engineers" コレクションに登録
6. `app.py`:登録件数をUIに表示

### マッチング実行フロー

1. `app.py`(メイン):入力テキストとボタン押下を受け取る
2. OpenAI Embeddings API:入力テキスト → クエリベクトル(1回のAPI呼び出し)
3. ChromaDB:コサイン類似度検索 → Top-3結果を返却
4. `chain.build_prompt()`:案件要件 + Top-3候補をプロンプトに埋め込む
5. OpenAI Chat Completions API:プロンプト → 理由テキスト生成
6. `app.py`:候補エンジニア情報と理由テキストをUIに表示

## 4. ChromaDBスキーマ設計

**コレクション名:** `engineers`

**フィールド構成**

| フィールド | 型 | 説明 |
|-------|------|-------------|
| id | str | エンジニアID(engineers.json から取得) |
| document | str | プロフィールテキスト(Embeddingの元テキスト) |
| embedding | vector | 1536次元 float(text-embedding-3-small) |
| metadata | dict | 補足フィールド(下記参照) |

**メタデータフィールド**

| キー | 型 | 説明 |
|-----|------|-------------|
| name | str | エンジニア名 |
| skills | str | スキル一覧(カンマ区切り文字列) |
| experience_years | float | 経験年数(8.5 のような端数を扱うため float で保持) |
| availability | str | 参画可能時期を自由形式の文字列として保持(例:「即日」「20260415」)。元フィールド:`参画可能時期`。 |

**永続化設定**

- ChromaDB `PersistentClient` を使用(`./chroma_db/` ディレクトリに保存)
- アプリ起動時に既存コレクションを自動ロード

## 5. プロンプト設計

**システムプロンプト**

```
You are an expert in SES matching.
Based on the job requirements and candidate engineer profiles provided,
explain with supporting evidence why each engineer is a good fit for
this job requirement.
Your response is intended to assist human decision-making and does not
guarantee hiring outcomes.
```

**ユーザープロンプトテンプレート**

```
[Job Requirements]
{job_text}

[Candidate Engineer List]
{candidates_text}

For each candidate engineer, explain in 2-3 bullet points (in Japanese)
how they align with the job requirements.
Use exactly this format (do not omit any [Candidate N] header):

[Candidate 1]
- reason
- reason

[Candidate 2]
- reason
- reason

[Candidate 3]
- reason
- reason
```

> `[Candidate N]` ヘッダーは必須。`chain.py` の `_parse_per_candidate_reasons` が正規表現で候補ごとに分割するためのアンカーとして利用する。

**`candidates_text` のフォーマット**

```
Candidate 1: {name}
Skills: {skills}
Experience: {experience_years} years
Similarity score: {score}
```

## 6. 環境変数と設定値

**環境変数(`.env` ファイル)**

| 変数名 | 必須 | 説明 |
|---------------|----------|-------------|
| `OPENAI_API_KEY` | 必須 | OpenAI APIキー |

**設定定数(コード内で定数として管理)**

| 定数名 | 値 | 説明 |
|---------------|-------|-------------|
| `EMBEDDING_MODEL` | `text-embedding-3-small` | 使用するEmbeddingモデル |
| `LLM_MODEL` | `gpt-4o-mini` | 使用するLLMモデル |
| `TOP_K` | `3` | 検索結果の上位件数 |
| `LLM_TEMPERATURE` | `0.3` | LLMの temperature パラメータ |
| `LLM_MAX_TOKENS` | `800` | LLMの最大トークン数 |
| `CHROMA_PERSIST_DIR` | `./chroma_db` | ChromaDB保存ディレクトリ |
| `COLLECTION_NAME` | `engineers` | コレクション名 |
| `ENGINEERS_DATA_PATH` | `./data/engineers.json` | エンジニアデータファイルパス |
